Targeting API Reference¶

class Targeting(datastore, clean_folders=False)¶
auc_overall(var1, var2, weighted=False, n_grid=99)¶

Computed the ROC-AUC scores at ‘n_grid’ equally spaced percentile thresholds.

Parameters

  • var1 (str) – The first variable to use in the analysis; it has to be a column in the targeting dataset.

  • var2 (str) – The second variable to use in the analysis; it has to be a column in the targeting dataset.

  • weighted (bool) – If True the weighted version of the dataset will be used.

  • n_grid (int) – The number of percentiles to compute the AUC scores for.

Return type

Dict[str, List[float]]

Returns: A dict with false/true positive rates and AUC scores for ‘n_grid’ percentiles between 0 and 100.

auc_threshold(var1, var2, p, t=None, weighted=False)¶

Computes the ROC-AUC score when the numerical variables are turned into binary using a certain threshold.

Parameters

  • var1 (str) – The first variable to use in the analysis; it has to be a column in the targeting dataset.

  • var2 (str) – The second variable to use in the analysis; it has to be a column in the targeting dataset.

  • p (Optional[Union[float, int]]) – The percentile to use to convert numerical values into binary.

  • t (Optional[Union[float, int]]) – The threshold to use to convert numerical values into binary.

  • weighted (bool) – If True the weighted version of the dataset will be used.

Return type

Dict[str, List[float]]

Returns: A dict with AUC score and false/true positive rate.

binary_metrics(var1, var2, p1, p2, t1=None, t2=None, weighted=False)¶

Computes several classification metrics for two numerical variables, usually a ground-truth one and its prediction; these are first converted into binary variables using the percentiles or thresholds provided.

Parameters

  • var1 (str) – The first variable to use in the analysis; it has to be a column in the targeting dataset.

  • var2 (str) – The second variable to use in the analysis; it has to be a column in the targeting dataset.

  • p1 (Union[int, float]) – Percentile for the first variable.

  • p2 (Optional[Union[float, int]]) – Percentile for the second variable.

  • t1 (Optional[Union[float, int]]) – Threshold for the first variable.

  • t2 (Optional[Union[float, int]]) – Threshold for the second variable.

  • weighted (bool) – If True the weighted version of the dataset will be used.

Return type

Dict[str, float]

Returns: A dictionary with results on accuracy, precision, recall, true and false positive rates.

precision_recall_curves(groundtruth, proxies, p=None, t=None, weighted=False, n_grid=99, colors=None)¶

Plots the ROC curves for all targeting methods specified by ‘proxies’, using the percentile/threshold provided OR a set of ‘n_grid’ equally spaced percentiles.

Parameters

  • groundtruth (str) – The name of the groundtruth column in the targeting dataset.

  • proxies (List[str]) – The list of targeting methods to be compared against each other; each name should have a corresponding column in the targeting dataset.

  • p (Optional[Union[float, int]]) – The percentile to use to convert numerical values into binary.

  • t (Optional[Union[float, int]]) – The threshold to use to convert numerical values into binary.

  • weighted (bool) – If True the weighted version of the dataset will be used.

  • n_grid (int) – The number of equally-spaced percentiles to compute the metrics for.

  • colors (Optional[Dict[int, str]]) – Color palette to use.

Return type

None

roc_curves(groundtruth, proxies, p=None, t=None, weighted=False, colors=None)¶

Plots the ROC curves for all targeting methods specified by ‘proxies’, using the percentile or threshold provided.

Parameters

  • groundtruth (str) – The name of the groundtruth column in the targeting dataset.

  • proxies (List[str]) – The list of targeting methods to be compared against each other; each name should have a corresponding column in the targeting dataset.

  • p (Optional[Union[float, int]]) – The percentile to use to convert numerical values into binary.

  • t (Optional[Union[float, int]]) – The threshold to use to convert numerical values into binary.

  • weighted (bool) – If True the weighted version of the dataset will be used.

  • colors (Optional[Dict[int, str]]) – Dict mapping proxy names to the color to use (in the default color palette)

Return type

None

targeting_table(groundtruth, proxies, p1, p2, t1=None, weighted=False)¶

Produces a table with the following metrics for all targeting methods specified by ‘proxies’: Pearson and Spearman correlation, threshold-specific and threshold-agnostic AUC, accuracy, precision, and recall.

Parameters

  • groundtruth (str) – The name of the groundtruth column in the targeting dataset.

  • proxies (List[str]) – The list of targeting methods to be compared against each other; each name should have a corresponding column in the targeting dataset.

  • p1 (Optional[Union[float, int]]) – The percentile threshold to use for the groundtruth variable.

  • p2 (Optional[Union[float, int]]) – The percentile threshold to use for the proxy variables.

  • t1 (Optional[Union[float, int]]) – The threshold to use for the groundtruth variable.

  • weighted (bool) – If True the weighted version of the dataset will be used.

Return type

pandas.core.frame.DataFrame

Returns: The pandas dataframe with all the performance metrics as columns and the proxies as rows.

static threshold_to_percentile(p, t, data, var, test_percentiles=True)¶

If a percentile is provided, checks if it’s between 0 and 100 and if so returns it; if a threshold is provided, returns the fraction of records in the dataset and for column ‘var’ that are below the threshold.

Parameters

  • p (Optional[Union[float, int]]) – The percentile.

  • t (Optional[Union[float, int]]) – The threshold.

  • data (pandas.core.frame.DataFrame) – The pandas dataframe containing the data.

  • var (str) – The column with the records to use to compute the percentile.

  • test_percentiles (bool) – If true checks that the percentile provided is correct.

Return type

Union[float, int]

Returns: The percentile corresponding to the input (threshold or percentile) provided.

utility(var1, var2, transfer_size, p, t=None, weighted=False, rho=3)¶

Computes the constant relative risk-aversion (CRRA) [Hanna & Olken (2018)] utility when P% of the population is targeted and they receive transfers of size ‘transfer_size’.

Parameters

  • var1 (str) – The first variable to use in the analysis; it has to be a column in the targeting dataset.

  • var2 (str) – The second variable to use in the analysis; it has to be a column in the targeting dataset.

  • transfer_size (Union[float, int]) – The size of the cash transfer.

  • p (Optional[Union[float, int]]) – The percentile to use to convert numerical values into binary.

  • t (Optional[Union[float, int]]) – The threshold to use to convert numerical values into binary.

  • weighted (bool) – If True the weighted version of the dataset will be used.

  • rho (Union[int, float]) – The rho parameter in the constant relative risk-aversion (CRRA) utility function.

Return type

float

Returns: The utility obtained by targeting that fraction of the population with that transfer size.

  • weighted (bool) – If True the weighted version of the dataset will be used.

  • n_grid (int) – The number of equally-spaced percentiles to compute the metrics for.

  • colors (Optional[Dict[int, str]]) – Color palette to use.

Return type

None

  • weighted (bool) – If True the weighted version of the dataset will be used.
